# Расширение root-раздела на полный размер SD карты

После прошивки `images/licheervnano.img` (2 GB) на SD карту большего объёма
неиспользованное пространство остаётся свободным. Эта инструкция расширяет
`/dev/mmcblk0p2` до полного размера карты.

Делается один раз после первого boot. Команды выполняются от root прямо на
плате через UART или ssh.

## Способ через fdisk

`fdisk` входит в base Debian, дополнительной установки не нужно.

### Шаг 1. (опционально) посмотреть текущий layout

```
fdisk -l /dev/mmcblk0
```

Пример вывода:

```
/dev/mmcblk0p1    2048    270335   268288  131M e W95 FAT16 (LBA)
/dev/mmcblk0p2  270336 ...        ...      ... 83 Linux
```

Start sector p2 в этой сборке всегда `270336` (сразу после p1).
p1 занимает 1MiB-132MiB и заканчивается на секторе 270335, поэтому при
пересоздании p2 fdisk сам предложит правильный default = 270336.

### Шаг 2. Запустить fdisk

```
fdisk /dev/mmcblk0
```

### Шаг 3. Удалить старый p2 и создать на его месте новый, до конца диска

Последовательно вводите команды fdisk (одна буква на строку):

```
d        delete partition
2        номер раздела
n        new partition
p        primary
2        номер раздела
         START sector = Enter (default 270336, сразу после p1)
         END = Enter (default до конца диска)
```

В этой сборке (с layout MBR=0, p1=1MiB-132MiB, p2=132MiB-end) fdisk
автоматически предлагает первый свободный sector = 270336 (после p1).
Default через Enter работает корректно.

На вопрос про сигнатуру файловой системы ответьте `N`:

```
Partition #2 contains a ext4 signature. Do you want to remove? N
```

Это критично: удаление сигнатуры разрушит существующий ext4 superblock
и приведёт к потере rootfs.

Сохранить и выйти:

```
w        write changes
```

После `w` fdisk обычно предупреждает что таблица меняется на mounted
устройстве. Это нормально.

### Шаг 4. Перечитать таблицу разделов

```
partprobe /dev/mmcblk0
```

Если `partprobe` не подхватил (ошибка BLKRRPART), просто `reboot`. После
перезагрузки таблица будет актуальной.

### Шаг 5. Расширить файловую систему

```
resize2fs /dev/mmcblk0p2
```

ext4 поддерживает online resize, размонтировать root не нужно.

### Шаг 6. Проверка

```
df -h /
```

Хочется увидеть размер близкий к полному размеру SD карты.

## Способ через parted (если установлен)

`parted` НЕ входит в EXTRA_PKGS этой сборки, нужно ставить отдельно:

```
apt install -y --no-install-recommends parted
```

Дальше одна строка вместо интерактивного fdisk:

```
parted -s /dev/mmcblk0 resizepart 2 100%
resize2fs /dev/mmcblk0p2
df -h /
```

`parted resizepart` автоматически сохраняет start sector, не требует
вводить его руками, и невозможно ошибиться с номером.

## Если что-то пошло не так

При следующем boot kernel panic `Unable to mount root fs` означает что
partition table указывает не на тот ext4 superblock. Восстановить можно
только перепрошивкой SD из `images/licheervnano.img` (на хосте):

```
sudo dd if=images/licheervnano.img of=/dev/sdX bs=4M status=progress conv=fsync
sync
```

Данные с p2 теряются. После перепрошивки повторите расширение
аккуратнее.

## Почему это не сделано автоматически

Можно добавить systemd-unit `resize-rootfs.service` который при первом
boot делает `growpart` (из cloud-guest-utils) и `resize2fs`, потом
disables себя. Это стандартная практика в Raspberry Pi OS, Armbian,
cloud images.

В этой сборке такого нет специально: расширение делается осознанно
один раз пользователем, и не происходит при каждом первом boot новой
SD карты (что было бы неожиданным для embedded use cases где образ
flash'ится на много карт).

Если хочется добавить автоматическое расширение, скажите и сделаем
отдельной коммитом.
